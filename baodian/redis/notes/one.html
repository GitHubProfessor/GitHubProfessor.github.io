# 2.值类型中hash和string有什么区别？实际工作中如何选择？
Redis hash 是一个 string 类型的 field 和 value 的 映射表。它的添加、删除操作都是 0(1)（平均操作）。
hash特别适合用于存储对象。相较于将对象的每个字段存成单个 string 类型（string 类型可以存储对象序列化）。 将一个对象存储在 hash 类型中会占用更少的内存，并且可以更方便的存取整个对象。 （省内存的原因是新建一个 hash 对象时开始是用 zipmap（又称为 small hash）来存储的。 这个 zipmap 其实并不是 hash table，但是 zipmap 相比正常的 hash 实现可以节省不少 hash 本身需要的一些元数据存储开销。
# 3.持久化是如何工作的？Redis在持久化过程中可以对外服务么？
我们都知道，redis是基于内存的K-V数据库。由于内存是断电易失的，所以redis提供了相应的持久化机制
持久化有两种方式：
1.rdb
2.aof
redis在持久化的过程中是可以对外服务的。
